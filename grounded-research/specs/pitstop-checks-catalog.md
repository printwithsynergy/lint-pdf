# Enfocus PitStop Pro - Preflight Checks Catalog

**Document**: Preflight Checks Overview (72 pages)
**Scope**: Complete catalog of every preflight check in Enfocus PitStop Pro, the industry-leading desktop preflight tool

---

## Overview

PitStop Pro provides comprehensive preflight checking across the following domains:

1. **PDF Standards** - PDF/X, PDF/A, PostScript fragments, output intents
2. **Document Properties** - Version, compression, encoding, security
3. **Pages** - Box layout, size, orientation, scaling, safety zones
4. **Transparency** - Blending, spot colors, overprinting
5. **Color** - RGB, ICC profiles, spot colors, ink coverage
6. **Rendering** - Transfer curves, halftones, rendering intents
7. **Fonts** - Type 1, TrueType, embedding, subset status
8. **Text** - Size, color visibility, invisibility
9. **Images** - Compression, resolution, OPI, rotation/scaling
10. **Layers** - Layer presence and print state
11. **Annotations** - Type, print settings, hyperlinks
12. **Other Objects** - JavaScript, forms, bookmarks, thumbnails
13. **Line Art** - Weight, visibility, nodes, flatness

## Implementation Feasibility Analysis

### High Feasibility Checks (pikepdf + Custom Engine)

These checks can be implemented directly using pikepdf's object inspection:

- **PDF Standards** (Section 4): Version key, output intent, default color spaces
- **Document Properties** (Section 5): PDF version, compression, encoding, security settings
- **Page Layout** (Section 6): Page boxes, size, orientation, scaling
- **Color Checks** (Section 8.1-8.8): Ink coverage, RGB detection, spot colors
- **Font Checks** (Section 10): Font type detection, embedding status, subsetting
- **Text Checks** (Section 11): Text size and visibility analysis
- **Image Checks** (Section 12): Compression type, resolution, transformation analysis

### Medium Feasibility Checks (Requires Additional Libraries)

These checks need external tools or advanced PDF processing:

- **Transparency** (Section 7): Requires blend mode inspection
- **ICC Profiles** (Section 8.15-8.23): Profile validation requires ICC library
- **Rendering Intent** (Section 9): Requires color management knowledge
- **Layer Analysis** (Section 13): Requires OCG (Optional Content Group) parsing

### Implementation Dependencies

| Check Type | Required Capability | Current Stack Support |
|---|---|---|
| PDF Validation | pikepdf object access | ✓ Full |
| Color Space Detection | PDF dictionary parsing | ✓ Full |
| Font Analysis | Font name/type inspection | ✓ Partial |
| Image Analysis | Stream inspection | ✓ Full |
| ICC Profile Validation | ICC parsing library | ✗ Not included |
| Transparency Analysis | Blend mode detection | ✓ Full |
| Security Inspection | Encryption dict parsing | ✓ Full |

## Grounded Integration Mapping

### Overlaps with PDF/X-4 Checks

The following PitStop checks align with our existing PDFX4-001 through PDFX4-092 checks:

- PS-001 (PDF/X Compliancy) → PDFX4-002 (PDF/X Version)
- PS-009 (Output intent) → PDFX4-015 (Output Intent)
- PS-010 (Default color spaces) → PDFX4-020 (Default Color Space)
- PS-018 (Color: RGB) → PDFX4-030 (No RGB)
- PS-031 (Ink coverage) → PDFX4-070 (Ink Density)
- PS-046 (Font not embedded) → PDFX4-050 (Font Embedding)

### Unique PitStop Checks (High Priority)

These checks are NOT in our PDFX4 set and should be prioritized:

1. **PS-008**: Spot color management (ambiguous names, bad suffixes)
2. **PS-011**: Image resolution validation
3. **PS-027**: Transparency with overprinting
4. **PS-052**: Safe type zone enforcement
5. **PS-085**: Rendering intent validation
6. **PS-095**: Halftone frequency analysis
