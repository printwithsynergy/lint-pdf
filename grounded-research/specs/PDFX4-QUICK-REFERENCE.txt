PDF/X-4 vs PDF/X-4p DIFFERENCES:
================================

PDF/X-4 (Complete Exchange):
  - REQUIRES embedded ICC profile in DestOutputProfile
  - All resources must be self-contained
  - File is complete and independent
  - Check: PDFX4-008 and PDFX4-089

PDF/X-4p (Partial Exchange with External Profile):
  - ICC profile can be external
  - OutputConditionIdentifier must uniquely identify profile
  - File depends on external profile availability
  - Checks: PDFX4-010, PDFX4-091, PDFX4-092

IMPLEMENTATION TIPS:
====================

1. Start with PDFX4-001 (PDF version) before any other checks
2. Verify PDFX4-003 (XMP metadata) early for conformance identification
3. Check PDFX4-005 and PDFX4-008 together (OutputIntent + ICC)
4. Cross-reference PDFX4-013 and PDFX4-014 for color space compliance
5. Use PDFX4-042 and PDFX4-044 to validate page geometry
6. Check PDFX4-054 and PDFX4-059 for security baseline
7. For PDF/X-4p variant, verify PDFX4-010 and PDFX4-091
8. Validate resource definitions (PDFX4-076-079) after content parsing

SPECIAL NOTES:
==============

Transparency: PDF/X-4 ALLOWS transparency (major difference from PDF/X-1a)
  - Validate structure but don't reject based on presence alone
  - Check soft masks (PDFX4-030) and group dictionaries (PDFX4-064)

Optional Content: PDF/X-4 ALLOWS layers (OCG)
  - Verify structure (PDFX4-056) but allow usage
  - Check intent definition (PDFX4-058)

Color Management: REQUIRES ICC profile
  - PDF/X-4: Embedded (PDFX4-008, 090)
  - PDF/X-4p: External reference (PDFX4-010, 091)
