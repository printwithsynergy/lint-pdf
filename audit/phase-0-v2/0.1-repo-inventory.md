## B. Engine (Python) entry points

**Path:** `packages/engine/`

### B.1 PDF parsing layer
**Library:** pikepdf 9 (QPDF wrapper)
**Wrapper:** `packages/engine/src/lintpdf/parser/`
- `adapter.py` — Abstract `ParserAdapter` interface
- `pikepdf_adapter.py` — Concrete impl (only file importing pikepdf directly)

The abstraction decouples analyzers from parsing library — allows future swap to pdfium/MuPDF without touching analyzers.

### B.2 Rasterization / rendering layer
**Libs:** Pillow 10, pdf2image 1.17 (wraps poppler/pdftoppm)
**Files:**
- `packages/engine/src/lintpdf/ai/rendering.py` — Tile rasterization for viewer
- `packages/engine/src/lintpdf/reports/page_renderer.py` — Report PDF rendering (HTML→PDF via WeasyPrint 62)

**Per-separation rendering:** `SeparationCanvas` and `SeparationRenderer` support CMYK/spot extraction.
**Coordinate system:** PDF points (1/72") at document layer, pixels at render time.

### B.3 ICC / color management

**Path:** `packages/engine/src/lintpdf/profiles/icc/`

**Device-link support:** None found. CxF spectral data parsing exists; full device-link chain absent — this is the gap blocking EPM-Advanced ΔE.

## L. Open observations / anomalies (15)

1. **Viewer architecture:** Canvas + pre-rendered tiles, not pdf.js or mupdf-wasm. Tight coupling between engine rendering (Pillow/pdf2image) and app viewer. Wave V will need to extend the tile pipeline or cache strategy.
2. **Tenant config is flat JSON** in `Tenant.settings`; no structured Prisma model. Type safety missing. v2 §16 needs a schema-enforced tenant_config Prisma model.
3. **Workflow concept absent.** Approval chains are NEW but no chaining/conditional/multi-stage templates.
4. **Desktop mirrors server config.** Cannot operate offline. v2 Wave V V-13 desktop sync is greenfield.
5. **No config-change audit trail.** v2 §16.5 audit logging is greenfield.
6. **NLP AI stubs not feature-gated by flag** — if `ai_features=['*']` reaches production, they may fail silently. Either gate or implement.
7. **No `remediation` field in FindingResponse** — v2 §7 contract gap. Operator must infer remediation from `message`.
8. **External format detection** by extension/hint only; no MIME-type sniffing. Mislabeled vendor reports may fail.
9. **ICC profile class detection exists; device-link application does NOT.** Blocker for EPM-Advanced ΔE.
10. **Two rendering pipelines** (pre-warm vs on-demand). Latency + cost implications. Consolidate or document.
11. **Color score breakdown** undocumented in OpenAPI; client interpretation unclear.
12. **Celery worker scaling** is manual replica scaling on Railway; no auto-scale to queue depth.
13. **Shared DB schema conflict** — Prisma + SQLAlchemy share one Postgres; `prisma db push --accept-data-loss` would delete engine tables. Mitigated by custom `startup.sh`.
14. **Pixie Dust dependency management** — caret-pinned in 3 packages; manual refresh on PD bumps; version skew possible.
15. **WS-17 Trim-box clipping** (`cropToTrim` in `PdfViewer.tsx`) implemented but off by default; v2 production-enablement decision needed.
