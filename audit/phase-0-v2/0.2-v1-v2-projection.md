## Methodology

The projection is **coarse, category-level**. Fine-grained ID-by-ID semantic
matching (which v2 ID corresponds to which engine emit-site) is reserved for
Phase 1 backlog — it requires reading every analyzer's emit logic. For
Phase 0 the goal is to identify *which v2 categories are credibly covered by
v1 work* and which are greenfield.

## Catalog overshoot — what it means

Three categories show >100% coverage (M, B, R). These are taxonomy-collapse
artifacts: v1's catalog buckets are broader than v2's prefix groups, so
multiple v1 categories map onto a single v2 prefix. Specifically:

- **M (Metadata) overshoot:** v1 `document` + `metadata` + `annotations` +
  `accessibility` + part of `structure` all collapse to M. v2 splits
  accessibility into its own concerns (touched by L for tag tree, M for
  XMP/marked content). Phase 1 backlog will need to re-segment.
- **B (Barcodes) overshoot:** v1 `barcodes` + AI `ai:barcode` subfamily
  (~25 IDs) collapse to v2's 28 B-* slots. v2 has fewer slots because some
  v1 barcode-content checks are categorized as `R` regulatory in v2. Re-segment.
- **R (Regulatory) overshoot:** All AI regulatory subbuckets (FDA, GHS,
  cosmetics, pharma, alcohol, cannabis, AFP, fibre, recycling) collapse into
  v2's 10 R-* slots. v2 represents regulatory as 10 high-level user-facing
  checks; the AI subdivision is internal. Re-segment.

**This means:** raw catalog coverage > 100% does NOT mean v1 covers v2's
needs in those categories. Fine-grained Phase 1 mapping will likely show
some v2 R-* / M-* / B-* checks are still absent despite the headline
overshoot.

## Recommendations for Phase 1 backlog

1. **Build the Tier-0 primitive registry first** (Wave 2.0, 3.0 EM). Every
   subsequent wave consumes primitives. Without a registry, each new check
   re-implements its predicate stack — code duplication and the absence of
   a uniform, testable foundation.
2. **Re-segment the 109 "other" catalog entries** during Phase 1 fine-grained
   ID mapping. Most are AI-tier; map them to v2's R-/F-/I-/D-/WF- prefixes
   per their semantic intent.
3. **Promote v1's dieline wedge work to v2's expanded D-/W-/F-/P- slots.**
   v1 closed all 15 of its dieline gaps; v2 expects 23 D-* slots plus
   dieline-adjacent F/P/L/T checks. Map v1 emit-sites to v2 IDs explicitly.
4. **Plan Wave V as a parallel track** from Wave B onward. The
   finding-emission contract is the prerequisite (§7 `remediation` field,
   geometry-locatable findings). Once stable, viewer overlay can evolve in
   parallel to subsequent check waves.
5. **EPM: ship Core first** (8 disqualifiers + 14 soft signals + scoring
   algorithm). Defer EPM-Advanced (device-link ΔE) until ICC engine is
   integrated. Defer EPM AI-Explain until per-tenant cost cap is wired.
6. **BYO §9: greenfield endpoint + schema validation + viewer hydration.**
   Distinct from existing import flow.
