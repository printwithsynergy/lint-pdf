## Cross-cutting risk: shared DB

Per CLAUDE.md §"Shared Database": app (Prisma) and engine (SQLAlchemy/Alembic)
share one Postgres. Wave V adds three new Prisma tables (Workflow,
LintPDFDecision, Toggle). Each needs:
1. Prisma `db push` (with `startup.sh` raw-SQL fallback for the engine-table conflict)
2. No `--accept-data-loss`
3. Migration tested against an engine-table-populated DB before prod

This is the single highest-risk operation in Wave V. **Recommend dry-run in
staging with full engine-table seed before prod migration.**
