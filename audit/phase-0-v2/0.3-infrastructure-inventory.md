## Critical infrastructure gaps (ranked)

1. **Tier-0 primitive registry (`packages/engine/src/lintpdf/primitives/`)** — 0/84 present. Foundation for every subsequent wave. **3.0 EM, blocks everything.**
2. **`remediation` field on `FindingResponse`** — additive schema change. Touches OpenAPI, every analyzer's emit, viewer drawer, every Phase 5 doc. **Should be the first non-Tier-0 change.**
3. **Workflow first-class entity** — pre-req for §16 cascade. Greenfield Prisma model + tRPC + UI + desktop sync. **Wave V V-12 + V-13 dependency.**
4. **Toggle registry + cascade resolver** — pre-req for any new toggle. **§16 hard requirement.**
5. **Lockable toggles** — pre-req for compliance-mandated checks. **§16.2.**
6. **Config audit log** — pre-req for compliance + debug. **§16.5.**
7. **Operator decision schema + viewer drawer** — Wave V V-03..V-05.
8. **Webhook idempotency key + outbox state** — Wave V V-06 hardening.
9. **Per-tenant theming for severity colors** — V-11.
10. **Device-link engine** — EPM-Advanced unblocker.
11. **BYO §9 schema endpoint** — Phase 2.EPM.5 / §9 / Wave V V-09.
12. **PDF/UA Matterhorn HTTP shim around veraPDF** — Wave C / Wave E.

## What v1 inventory got right

The engine is well-structured: pikepdf decoupled via `ParserAdapter`, AI
analyzers in their own subtree, conformance validators isolated, Celery
workers split by concern (Worker / Worker-AI / Worker-Reports / Worker-Tiles
/ Worker-Webhooks). The 38-analyzer registry pattern is clean. v1 work
inherits and extends this — no architectural debt to undo.

## What needs greenfield (in priority order)

1. `primitives/` — primitive registry
2. `epm/` — Phase 2.EPM module (Core / Advanced / AI-Explain)
3. `byo/` — §9 schema validation + endpoint
4. App: `Workflow` Prisma model + tRPC router + desktop sync API
5. App: `Toggle` registry + resolver + audit-log middleware
6. App: `LintPDFDecision` table for per-finding operator decisions
7. App: viewer drawer component + per-finding click event
8. App: theming system for severity colors
9. App: webhook outbox state + idempotency-key support
10. Desktop: reconcile-on-reconnect protocol + offline read mirror
