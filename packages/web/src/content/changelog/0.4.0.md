### Added

- **Plan-included monthly AI credits** — every paid plan now grants a monthly bucket of AI-inspection credits that resets each billing cycle. Free: 0 · Viewer: 0 · Starter: 100 · Growth: 500 · Scale: 2,000 · Enterprise: 10,000.
- **Plan-included monthly file allotments** — each paid plan also includes a monthly file quota on top of the existing daily rate limit. Free: 50 · Viewer: 50 · Starter: 500 · Growth: 2,500 · Scale: 10,000 · Enterprise: 100,000.
- **Top-up credit packs** — three Stripe Checkout SKUs (500 / 2,000 / 10,000 credits at $25 / $90 / $400) with tiered discounts. Purchased packs roll over for 12 months.
- **Top-up file packs** — parallel SKUs (500 / 2,500 / 10,000 files at $15 / $60 / $200) for submissions beyond the plan allotment.
- **Dashboard billing pages** — `Account → AI Credits` and `Account → File Packs` surface current balance, monthly allotment, purchase history, and one-click Stripe Checkout purchase flows.
- **Admin Metered Resources console** — super admins can override monthly allotments per tenant, grant credits/files directly (`source=admin_grant`, no Stripe charge), and revoke individual packages from a new `/dashboard/admin/billing` page.
- **Engine endpoints** — new `POST /api/v1/ai/credits/topup` (now returns a Stripe Checkout URL), `POST /api/v1/files/topup`, `GET /api/v1/files/quota`, `GET /api/v1/ai/credits/packages`, `GET /api/v1/files/packages`, plus admin monthly-override and package-revoke routes. Stripe webhook fulfillment via `POST /api/v1/billing/stripe/webhook` (signature-verified).
- **Webhook-driven monthly grants** — `invoice.paid` now automatically allocates plan-monthly packs for both kinds. Idempotent on `(tenant, kind, billing_period_start)` so replays never double-grant.
- **Idempotency for purchases** — every pack row has a unique `stripe_session_id` so duplicate `checkout.session.completed` webhook deliveries return the existing pack instead of creating duplicates.

### Changed

- `POST /api/v1/ai/credits/topup` request shape — legacy `{ "credits": N }` is replaced with `{ "pack": "500" | "2000" | "10000" }`. The server resolves the Stripe price ID from the pack size.
- Every paid-plan `invoice.paid` event now fires metered-resource allocation in addition to subscription sync.
- Daily rate limit still enforced as an abuse shield alongside the new monthly file quota.

### Migration

- Alembic `027_metered_resource_packs` (additive + defaulted): adds `kind`, `source`, `stripe_session_id`, `billing_period_start` to `tenant_ai_credit_packages` and `monthly_ai_credits_override` + `monthly_files_override` to `tenants`. Safe on tables with existing rows.
- No breaking changes for existing customers — the engine falls back to legacy daily-only file limiting until `invoice.paid` seeds the first monthly pack, so nothing 402s on the upgrade cutover.

### Docs

- Rewrote [AI Credits](/docs/ai-credits) with the new pack pricing, monthly allotment table, and Stripe Checkout flow.
- Pricing page now shows the plan-included allotment table plus top-up packs for both credits and files.
