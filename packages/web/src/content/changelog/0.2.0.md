### Added

- **Livery (white-label branding)** — Scale and Enterprise plans can now configure custom logos, colors, and footer text for PDF reports via `PUT /api/v1/livery`.
- **Non-PDF input support** — Check-In now accepts EPS, PostScript, TIFF, JPEG, PNG, and PDF-compatible AI files. Non-PDF files are converted internally before Inspection.
- **XML report format** — Flight Logs can now be downloaded as XML via the `format=xml` query parameter on the report endpoint.

### Changed

- Report endpoint path changed from `/api/v1/jobs/{id}/reports` to `/api/v1/flight-log/{id}/report` to align with branded terminology.
- API key prefix changed from `grd_` to maintain consistency across all key types.

### Fixed

- Corrected PDF/A validation for documents with embedded JavaScript that should trigger a No-Fly finding.
- Resolved timeout issues for files larger than 200 MB during transparency analysis.
