## Issues found (every one gets fixed)

### ISSUE-1 [major] annotated PDFs only overlay findings that have bbox (10/266 = 4%)

Only 10 of 266 findings in the sample job carry a bbox; the annotated_pdf format's per-page content stream is nearly identical to the source PDF. Most checks (AI_WCAG_001, LPDF_TEXT_004, LPDF_INK_002 etc.) know the page and font but aren't emitting bbox to FindingResponse. Needs propagation through the analyzers that already compute geometry.

**Files:**
- `packages/engine/src/lintpdf/api/schemas.py`
- `packages/engine/src/lintpdf/analyzers/`
- `packages/engine/src/lintpdf/reports/annotated_pdf_report.py`

### ISSUE-2 [major] annotated_pdf_markup silently dropped from reports response when tenant has no viewer annotations

POST /jobs/{id}/reports with formats=[...,'annotated_pdf_markup'] returns 201 with the markup entry missing from the response instead of an entry with data=None / content_type=null and an explanation. Caller can't distinguish 'format intentionally skipped' from 'format failed'. See service.py:1235-1243.

**Files:**
- `packages/engine/src/lintpdf/reports/service.py`

### ISSUE-3 [major] error message cites wrong plan when annotated PDF denied

403 response says 'required_plan: starter' for annotated_pdf, but the actual plans that allow annotated_pdf are scale + enterprise (tenants/models.py:125-132, 155-162). starter and growth do NOT include it. The error misleads customers into a cheaper plan that won't fix the problem.

**Files:**
- `packages/engine/src/lintpdf/api/routes/reports.py`
- `packages/engine/src/lintpdf/tenants/models.py`

### ISSUE-4 [moderate] PUT /admin/tenants/{id}/ai returns 500 on invalid billing_mode

Passing billing_mode=unlimited (not in AIBillingMode enum) caused HTTP 500 Internal Server Error. Should validate against the enum and return 422 with allowed values.

**Files:**
- `packages/engine/src/lintpdf/api/routes/admin.py`

### ISSUE-5 [moderate] AI credit_balance field stays at 0 after admin grant

POST /admin/tenants/{id}/ai/credits returns 200 with credits_granted: 1000, but GET /admin/tenants/{id}/ai still shows credit_balance: 0.0. Either the balance aggregation is wrong or the grant didn't link to the AI billing row.

**Files:**
- `packages/engine/src/lintpdf/ai/config.py`
- `packages/engine/src/lintpdf/api/routes/admin.py`

### ISSUE-6 [moderate] JobStatus enum uses 'complete' but API docs/tests expect 'completed'

Polling jobs requires matching both 'complete' (enum value) and 'completed' (common-sense naming). The API reference page and first-run driver code expect 'completed' and silently loop forever. Normalize one direction.

**Files:**
- `packages/engine/src/lintpdf/api/models.py`

### ISSUE-7 [moderate] GPU inference service unavailable in prod (AI_AFP_001 advisory)

Preflight emits 'AI_AFP_001: GPU inference service unavailable for preflight profile recommendation' for every full-ai-scan job in prod. Whole AI categories (image quality, logo detection, dieline, NSFW, profile recommendation) silently degrade to 1 finding each. Either wire up the GPU inference service on Railway or demote these from full-ai-scan so customers don't pay credits for skipped checks.

**Files:**
- `packages/inference/`
- `packages/engine/src/lintpdf/ai/analyzers/`

### ISSUE-8 [minor] AI + AI-credits endpoints blanket-denied with 'invite-only alpha' when tenant.ai_enabled=false

Fresh tenants (any plan) get 403 on /ai/credits, /ai/usage, /ai/config PUT unless admin toggles tenant.ai_enabled=true. The error copy 'invite-only alpha' is misleading — it's a per-tenant flag, not a global one. Either auto-enable on growth+ plans or update the copy.

**Files:**
- `packages/engine/src/lintpdf/ai/access.py`
